## Gotcha: Cloudflare "Connect GitHub" dead-end
- Clicking "Connect GitHub" in the Pages setup wizard opened the GitHub OAuth consent screen normally
- After authorising, the return redirect to Cloudflare did **not** register the connection — UI sat as if nothing had happened
- **Workaround**: manually navigate to `https://dash.cloudflare.com/<account-id>/pages/new/provider/github/authorize` to complete the OAuth handshake
- Once done, the connected repos appeared in the wizard normally
- Worth documenting for anyone else hitting this — appears to be a Cloudflare UI bug, not a config issue on our side

## Lessons learned / process notes
- The build-first check (`npm run build` locally before connecting Cloudflare) paid off. Caught nothing broken but gave confidence the Cloudflare build would succeed.
- `public/_redirects` is a one-liner but a blocker without it. Worth capturing as the first thing to add when deploying any SPA to Cloudflare Pages.
- Cloudflare's UI glitch ate ~15 minutes of confusion. The direct `/pages/new/provider/github/authorize` URL is the workaround to remember.
- User-driven dashboard setup interleaved with Claude Code's code prep worked well — clear hand-off points and each side could act without waiting.
